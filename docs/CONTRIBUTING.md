# Contributing

## 기본 절차

1. 이슈를 생성하거나 기존 이슈를 할당받습니다.
2. 기능 브랜치를 생성합니다.
3. 변경 후 `pnpm run verify:scaffold`를 실행해 기본 구조를 확인합니다.
4. Pull Request를 생성하고 템플릿을 채워 리뷰를 요청합니다.

## 커밋 규칙

- 가능한 한 작은 단위로 커밋합니다.
- 변경 의도를 커밋 메시지에 명확히 작성합니다.

## Pull Request 체크리스트

- 변경 배경 및 목적
- 주요 변경 사항
- 테스트/검증 결과
- 리뷰어가 확인할 포인트
